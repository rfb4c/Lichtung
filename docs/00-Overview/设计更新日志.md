## v0.2.0 — 用户系统 + 评论区 (2026-02-17)

### 新增功能

**用户认证系统**
- Supabase Auth 注册/登录（邮箱 + 密码）
- 用户资料表（profiles），预留 Path C 身份字段：城市、职业、兴趣标签
- 注册时自动创建 profile（PostgreSQL trigger）
- 侧边栏用户菜单（UserMenu）：登录按钮 / 已登录头像+昵称
- 个人资料编辑器（ProfileEditor）：编辑昵称、城市、职业、兴趣
- AuthModal：Twitter 风格的全屏登录/注册覆盖层

**评论区系统**
- 展开式评论（点击 FeedItem 评论图标展开/收起）
- CommentSection：加载评论列表 + 评论输入
- CommentItem：头像 · 昵称 · 城市 · 职业 · 时间 + 评论文本
- CommentInput：已登录显示输入框，未登录显示登录提示
- 评论计数：Supabase 实时获取，fallback 到伪随机数

**基础设施**
- 提取 mappers.ts：集中管理 Supabase 行类型到前端类型的映射
- 提取 config.ts：共享 isSupabaseConfigured 判断
- AuthContext：全局认证状态管理

### 对应研究路径
- **路径 C 基础**：profiles 表的 city/profession/interests 字段为"交叉身份线索"奠定数据基础
- **评论区**：为后续在评论中展示身份线索提供界面载体

### 数据库变更
- 新增 `profiles` 表（id, display_name, avatar_url, city, profession, interests）
- 新增 `comments` 表（id, report_id, user_id, content, created_at）
- 新增 `handle_new_user` 触发器（注册自动创建 profile）
- RLS 策略：profiles/comments 公开读取，写入需认证

### 文件变更
- 新建 17 个文件（组件、样式、Context、工具模块、文档）
- 修改 5 个文件（types, main.tsx, App.tsx, FeedItem.tsx, Sidebar.tsx）

### 已知问题
- **注册错误 "Database error saving new user"**：触发器 `handle_new_user` 中未显式指定 `public` schema，导致 Supabase 在 auth schema 上下文中找不到 profiles 表。修复方案：触发器函数加 `SET search_path = public`，INSERT 目标改为 `public.profiles`
- **登录错误**：待排查（v0.2.0 已知，待修复）

### 待办

## v0.2.1 — 个人资料页面 + 空表回退 (2026-02-18)

### 新增功能

**个人资料独立页面**
- 将 ProfileEditor 从弹出框（popup）重构为独立页面视图
- 侧边栏点击"个人资料"导航到专属页面，而非弹出编辑框
- Twitter 风格个人资料页：头部 banner + 头像 + 昵称 + 个人信息 + 编辑按钮
- 个人资料页展示"我的评论"列表（替代"我的帖子"）
  - 设计决策：Lichtung 是新闻 Feed 阅读平台，用户不发帖，评论是唯一的内容产出
  - 每条评论关联回原报道，显示"评论了《XXX》"上下文
  - 无评论时显示空状态提示

**数据空表回退**
- Supabase events/reports 表为空时，自动回退到静态 JSON（events.json）数据

### 对应研究路径
- **路径 C 基础**：个人资料页面为身份线索展示（城市、职业、兴趣）提供独立空间
- **评论历史**：用户评论记录本身体现立场和关注点，为后续研究分析提供数据

### 文件变更
- 重构 `src/components/ProfileEditor.tsx` → 独立页面组件（ProfilePage）
- 修改 `src/components/Sidebar.tsx` — 页面导航替代弹出框
- 修改 `src/App.tsx` — 添加页面路由/视图切换、空表回退逻辑

### 已知问题

### 待办

## v0.1.0 — Feed 流 + 光谱可视化 (2026-02-14)

### 功能
- Twitter 风格 Feed 流（3 列布局：侧边栏 + 600px 主 Feed + 右侧栏）
- 事件 + 报道数据模型（3 个事件，29 篇报道）
- SpectrumBar 光谱条（路径 B：共识数据可视化）
- 悬停 500ms 延迟显示分布 Tooltip，300ms 淡出
- 深色/浅色主题切换（localStorage 持久化）
- Supabase 集成（events + reports 表），回退到静态 JSON
- 报道卡片：图片、来源标识、立场标签、链接卡片
- 思源宋体 + 文字 Logo
- 自定义 Favicon
- Lucide 图标库

### 数据库
- `events` 表（id, title, supportive, neutral, opposed）
- `reports` 表（id, event_id, title, summary, source, stance, image_url, published_at）
- RLS 策略：公开只读
