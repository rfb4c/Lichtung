## 1. 系统架构

### 1.1 当前 Demo 版本架构（v0.2.x）

```
手动准备的静态数据（JSON 文件或手动填充到 Supabase）
    │
    ▼
Supabase (PostgreSQL + REST API)
    │
    │  @supabase/supabase-js
    │  （优雅降级：Supabase 未配置时回退到静态 JSON）
    │
    ▼
前端 (Vite + React + TypeScript)
    │
    └─ 用户认证（Supabase Auth）
    └─ 评论系统（已实现）
    └─ Path B 可视化（待实现）
    └─ Path C 身份标签（待实现）
```

**核心特点**：
- ✅ **数据写死**：报道、民调数据、用户标签均为手动准备的 mock 数据
- ✅ **无自动化管线**：不实现 NewsAgent，不自动抓取报道
- ✅ **适合 Demo**：快速验证干预机制，向导师展示概念原型

### 1.2 未来扩展版本架构（规划，不在当前实现）

```
NewsAgent (自动新闻采集)
    │
    │  打议题标签、抓取报道
    │
    ▼
Supabase (PostgreSQL + REST API)
    │
    │  @supabase/supabase-js
    │
    ▼
前端 (Vite + React + TypeScript)
```

**扩展方向**（详见 [03-News-Agent 设计规范](../03-News-Agent/设计规范.md)）：
- 自动抓取新闻报道
- 自动打议题标签
- 定时更新数据

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vite + React 18 + TypeScript | 已实现基础 MVP |
| 数据库 | Supabase (PostgreSQL) | 托管服务，免费额度够用 |
| 认证 | Supabase Auth | 邮箱密码登录，已实现 |
| 数据来源（当前）| 手动准备的 JSON 或手动填充 | Demo 阶段写死数据 |
| 数据来源（未来）| NewsAgent + Pew Research API | 自动化扩展方向 |

### 2.6 建表 SQL（完整版本）

```sql
-- ========== 基础数据表（已实现） ==========

CREATE TABLE events (
  id text PRIMARY KEY,
  title text NOT NULL,
  supportive integer NOT NULL,
  neutral integer NOT NULL,
  opposed integer NOT NULL,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE reports (
  id text PRIMARY KEY,
  event_id text NOT NULL REFERENCES events(id),
  title text NOT NULL,
  summary text NOT NULL,
  source text NOT NULL,
  stance text NOT NULL,
  topic_id text,  -- 新增，关联 topics 表
  image_url text,
  published_at text,
  created_at timestamptz DEFAULT now()
);

CREATE INDEX idx_reports_event_id ON reports(event_id);
CREATE INDEX idx_reports_stance ON reports(stance);

-- ========== 用户系统表（已实现，v0.2.0） ==========

CREATE TABLE profiles (
  id uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  display_name text,
  avatar_url text,
  city text,
  profession text,
  interests text[],
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

CREATE TABLE comments (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  report_id text NOT NULL REFERENCES reports(id) ON DELETE CASCADE,
  user_id uuid NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  content text NOT NULL,
  created_at timestamptz DEFAULT now()
);

CREATE INDEX idx_comments_report_id ON comments(report_id);
CREATE INDEX idx_comments_user_id ON comments(user_id);

-- ========== Path B 数据表（待实现） ==========

CREATE TABLE topics (
  id text PRIMARY KEY,
  name text NOT NULL,
  scope text NOT NULL,
  tag_keywords text[],
  created_at timestamptz DEFAULT now()
);

CREATE TABLE polling_data (
  id text PRIMARY KEY,
  topic_id text NOT NULL REFERENCES topics(id) ON DELETE CASCADE,
  source text NOT NULL,
  survey_year integer NOT NULL,
  geographic_scope text NOT NULL,
  scale_labels text[] NOT NULL,
  distribution integer[] NOT NULL,
  bridging_text text NOT NULL,
  created_at timestamptz DEFAULT now()
);

CREATE INDEX idx_polling_topic_id ON polling_data(topic_id);

-- 添加 reports.topic_id 外键约束
ALTER TABLE reports ADD CONSTRAINT fk_reports_topic
  FOREIGN KEY (topic_id) REFERENCES topics(id);

-- ========== Path C 数据表（待实现） ==========

CREATE TABLE tags (
  id text PRIMARY KEY,

CREATE TABLE user_tags (

  tag_id text NOT NULL REFERENCES tags(id) ON DELETE CASCADE,

  created_at timestamptz DEFAULT now(),

CREATE INDEX idx_user_tags_user_id ON user_tags(user_id);
CREATE INDEX idx_user_tags_tag_id ON user_tags(tag_id);
```

## 3. 数据流

### 3.1 当前 Demo 版本数据流（v0.2.x）

```
手动准备 Mock 数据
    │
    ├─ 方式 A：写死在前端 JSON 文件中（src/data/*.json）
    │   └─ 优雅降级：Supabase 未配置时使用
    │
    └─ 方式 B：手动填充到 Supabase 数据库
        └─ 通过 Supabase Dashboard SQL Editor 直接 INSERT
        └─ 或通过 Node 脚本调用 Supabase API 批量插入
```

**核心原则**：
- ✅ **数据完全手动准备**：不依赖自动化采集
- ✅ **优雅降级**：Supabase 未配置时回退到静态 JSON
- ✅ **快速验证**：专注干预机制本身，不在数据采集上耗时

### 3.2 Mock 数据准备清单（Demo 阶段）

| 数据类型 | 数量 | 准备方式 | 数据来源 |
|---------|------|---------|---------|
| **events** | 3-5 个事件 | 手动编写 JSON | 基于真实新闻事件改写 |
| **reports** | 每个事件 5-10 篇报道 | 手动编写 JSON | 基于真实新闻标题改写 |
| **topics** | 2-3 个议题 | 手动编写 JSON | Path B 设计规范指定（枪支、堕胎、气候） |
| **polling_data** | 每议题 1 条民调 | **手动验证后录入** | **严格使用 Pew Research 原始数据** |
| **tags** | 30-50 个标签 | 手动编写 JSON | Path C 设计规范的四层标签体系 |
| **user_tags** | 10 个 mock 用户，每人 3-5 个标签 | 手动编写 JSON | 确保存在交叉身份场景 |
| **comments** | 每报道 5-10 条评论 | 手动编写 JSON | **故意偏向极端**，模拟多数错觉 |

**重要说明**：
- `polling_data` 必须使用真实权威数据，不可编造
- `comments` 应故意选取极端立场，以制造与民调数据的对比效应

### 3.4 前端数据加载（优雅降级）

```typescript
import { createClient } from '@supabase/supabase-js'
import { isSupabaseConfigured } from './lib/config'

// 静态 JSON 作为回退数据
import eventsData from './data/events.json'
import reportsData from './data/reports.json'

const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY)

// App.tsx 中
const [events, setEvents] = useState<Event[]>([])
const [reports, setReports] = useState<Report[]>([])
const [loading, setLoading] = useState(true)

useEffect(() => {
  async function fetchData() {
    if (isSupabaseConfigured) {
      // 从 Supabase 加载
      const [eventsRes, reportsRes] = await Promise.all([
        supabase.from('events').select('*'),
        supabase.from('reports').select('*'),
      ])
      if (eventsRes.data) setEvents(eventsRes.data)
      if (reportsRes.data) setReports(reportsRes.data)
    } else {
      // 回退到静态 JSON
      setEvents(eventsData)
      setReports(reportsData)
    }
    setLoading(false)
  }
  fetchData()
}, [])
```

### 5.2 安全策略（RLS）

```sql
-- ========== 基础数据表（公开只读） ==========
ALTER TABLE events ENABLE ROW LEVEL SECURITY;
ALTER TABLE reports ENABLE ROW LEVEL SECURITY;

CREATE POLICY "events_public_read" ON events FOR SELECT USING (true);
CREATE POLICY "reports_public_read" ON reports FOR SELECT USING (true);

-- ========== 用户系统表（已实现） ==========
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE comments ENABLE ROW LEVEL SECURITY;

CREATE POLICY "profiles_public_read" ON profiles FOR SELECT USING (true);
CREATE POLICY "profiles_update_own" ON profiles FOR UPDATE USING (auth.uid() = id);

CREATE POLICY "comments_public_read" ON comments FOR SELECT USING (true);
CREATE POLICY "comments_insert_authenticated" ON comments FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- ========== Path B 数据表（待实现） ==========
ALTER TABLE topics ENABLE ROW LEVEL SECURITY;
ALTER TABLE polling_data ENABLE ROW LEVEL SECURITY;

CREATE POLICY "topics_public_read" ON topics FOR SELECT USING (true);
CREATE POLICY "polling_data_public_read" ON polling_data FOR SELECT USING (true);

-- ========== Path C 数据表（待实现） ==========
ALTER TABLE tags ENABLE ROW LEVEL SECURITY;
ALTER TABLE user_tags ENABLE ROW LEVEL SECURITY;

CREATE POLICY "tags_public_read" ON tags FOR SELECT USING (true);
CREATE POLICY "user_tags_public_read" ON user_tags
  FOR SELECT USING (is_public = true);
CREATE POLICY "user_tags_manage_own" ON user_tags
  FOR ALL USING (auth.uid() = user_id);
```

   - **严格核对**量表档数和各档百分比
   - 录入到 `src/data/polling_data.json`
4. **手动打议题标签**：
   - 为现有报道（reports）手动添加 `topic_id` 字段
   - 或创建新的报道数据，包含议题标签
5. **导入到 Supabase**：
   - 方式 A：Supabase Dashboard SQL Editor 直接 INSERT
   - 方式 B：Node 脚本批量插入
   - 或保持数据在前端 JSON，不导入 Supabase（优雅降级）
