## 1. 系统架构

```
Gemini Deep Research (手动运行)
    │
    │  输出结构化 JSON
    │
    ▼
Supabase (PostgreSQL + REST API)
    │
    │  @supabase/supabase-js
    │
    ▼
前端 (Vite + React + TypeScript)
```

### 1.1 架构说明

- **数据采集**：通过 Gemini Deep Research 手动运行，输出符合约定 schema 的 JSON
- **数据存储**：Supabase 托管 PostgreSQL，自带 REST API，无需自建后端
- **前端展示**：React SPA，通过 Supabase client 直接查询数据库

### 1.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vite + React 18 + TypeScript | 已有 MVP |
| 数据库 | Supabase (PostgreSQL) | 托管服务，免费额度够用 |
| 数据采集 | Gemini Deep Research | 手动触发，输出 JSON |

## 2. 数据库设计

### 2.1 events 表

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | text | PK | 如 `ev_001` |
| title | text | NOT NULL | 事件名称 |
| supportive | integer | NOT NULL | 支持占比 |
| neutral | integer | NOT NULL | 中立占比 |
| opposed | integer | NOT NULL | 反对占比 |
| created_at | timestamptz | DEFAULT now() | 创建时间 |

### 2.2 reports 表

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | text | PK | 如 `rp_001` |
| event_id | text | FK → events.id | 关联事件 |
| title | text | NOT NULL | 报道标题 |
| summary | text | NOT NULL | 摘要 |
| source | text | NOT NULL | 来源 |
| stance | text | NOT NULL | `supportive` / `neutral` / `opposed` |
| image_url | text | | 配图 URL |
| published_at | text | | 发布时间 |
| created_at | timestamptz | DEFAULT now() | 创建时间 |

### 2.3 建表 SQL

```sql
CREATE TABLE events (
  id text PRIMARY KEY,
  title text NOT NULL,
  supportive integer NOT NULL,
  neutral integer NOT NULL,
  opposed integer NOT NULL,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE reports (
  id text PRIMARY KEY,
  event_id text NOT NULL REFERENCES events(id),
  title text NOT NULL,
  summary text NOT NULL,
  source text NOT NULL,
  stance text NOT NULL,
  image_url text,
  published_at text,
  created_at timestamptz DEFAULT now()
);

CREATE INDEX idx_reports_event_id ON reports(event_id);
CREATE INDEX idx_reports_stance ON reports(stance);
```

## 3. 数据流

### 3.1 数据导入流程

```
1. 运行 Gemini Deep Research，使用约定 prompt
2. Gemini 输出 JSON（格式见 3.2）
3. 导入 Supabase：
   - 方式 A：Supabase Dashboard SQL Editor 直接插入
   - 方式 B：Node 脚本调用 Supabase API 批量插入
```

### 3.2 Gemini 输出 JSON 格式

```json
{
  "events": [
    {
      "id": "ev_001",
      "title": "事件名称",
      "supportive": 30,
      "neutral": 45,
      "opposed": 25
    }
  ],
  "reports": [
    {
      "id": "rp_001",
      "event_id": "ev_001",
      "title": "报道标题",
      "summary": "报道摘要（100-200字）",
      "source": "国内",
      "stance": "supportive",
      "image_url": "https://...",
      "published_at": "2小时前"
    }
  ]
}
```

### 3.3 前端数据加载

```typescript
// 当前：静态导入
import eventsData from './data/events.json'

// 目标：从 Supabase 查询
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY)

// App.tsx 中
const [events, setEvents] = useState<Event[]>([])
const [reports, setReports] = useState<Report[]>([])
const [loading, setLoading] = useState(true)

useEffect(() => {
  async function fetchData() {
    const [eventsRes, reportsRes] = await Promise.all([
      supabase.from('events').select('*'),
      supabase.from('reports').select('*'),
    ])
    if (eventsRes.data) setEvents(eventsRes.data)
    if (reportsRes.data) setReports(reportsRes.data)
    setLoading(false)
  }
  fetchData()
}, [])
```

## 5. Supabase 配置

### 5.1 项目创建

1. 注册 Supabase 账号
2. 创建新项目
3. 在 SQL Editor 中执行 2.3 节建表 SQL
4. 在 Settings → API 中获取 `URL` 和 `anon key`

### 5.2 安全策略（RLS）

```sql
-- 前端只需读取，启用 RLS 后添加公开读权限
ALTER TABLE events ENABLE ROW LEVEL SECURITY;
ALTER TABLE reports ENABLE ROW LEVEL SECURITY;

CREATE POLICY "events_public_read" ON events
  FOR SELECT USING (true);

CREATE POLICY "reports_public_read" ON reports
  FOR SELECT USING (true);
```

### 5.3 环境变量

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```
