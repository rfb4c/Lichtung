# 林间空地 Lichtung - 产品设计文档

> **产品名称**：林间空地 Lichtung**命名寓意**：海德格尔哲学中的"澄明之境"——在算法密林中显现真实的空地**项目定位**：研究用Demo，验证"多数错觉可视化"干预对用户感知极化的校准效果**受众**：下一阶段的实施团队（AI或人类开发者）**时间估算**：3-4天完成MVP

* * *

## 1. 背景与研究动机

### 1.1 核心问题

**多数错觉（Majority Illusion）**：

* Lerman et al. (2016)在*PLOS ONE*发现：社交网络的异配结构中，仅20%的节点活跃即可导致60-70%的节点产生"该观点为多数"的错误感知
* 算法推荐系统放大这一效应：高参与度内容（点赞、转发、评论）被过度呈现，用户误以为"我看到的=真实分布"

**现实影响**：

* 用户基于偏差样本形成对"外群体立场"的误判
* 夸大对方敌意程度（Moore-Berg et al., 2020）
* 形成"大家都很激进"的错觉

### 1.2 干预假设

**如果**直接展示真实分布数据，打破用户基于局部观察的推断**那么**用户对意见分布的估计将更准确，从而降低感知极化

### 1.3 设计目标

**不是**：

* ❌ 改变用户的政治立场
* ❌ 推荐"更好"的内容
* ❌ 做完整的新闻产品

**而是**：

* ✅ 让用户意识到"我看到的≠全部"
* ✅ 校准用户对意见分布的感知
* ✅ 验证"分布可视化"的干预效果

## 2. 产品设计方案

### 2.1 核心功能

**Feed流 + 悬停显示分布**

**设计理念**：

* 体验式学习（Experiential Learning）：让用户自己发现预期与现实的差异
* 轻推（Nudge）：非侵入式干预，不打断阅读流程
* 认知失调（Cognitive Dissonance）：触发"我以为vs实际"的冲击

### 2.2 用户旅程

    用户打开网站
        ↓
    滑动Feed流，浏览多个事件的报道
        ↓
    [可能] 形成印象："哇，好多人在反对这个政策"
        ↓
    悬停到某篇报道
        ↓
    看到光谱条：反对 25% ← 此报道
        ↓
    [认知冲击] "什么？只有25%？我刚才明明看到很多反对的..."
        ↓
    [自我发现] "原来我看到的不是全貌"

## 5. 实施步骤

### Phase 1: 数据准备（半天）

**任务**：直接生成模拟数据，无需手工收集

**AI自动生成数据的Prompt**：

    请为"林间空地"新闻阅读器生成模拟数据。
    
    要求：
    1. 创建2-3个热点事件（如"禁燃油车政策"、"AI管理新规"等）
    2. 每个事件生成20-30篇模拟报道
    3. 报道分布应符合"多数错觉"场景：
       - 真实分布相对均衡（如：支持25%、中立50%、反对25%）
       - 但"反对"类报道的标题更吸睛、更容易被转载
    4. 每篇报道包括：标题、摘要、来源（外媒/国内）、立场
    
    输出JSON格式：
    {
      "events": [...],
      "reports": [...]
    }
    
    参考数据结构见文档4.3节。

**生成策略**：

* 标题要真实可信（不要太假）
* 分布数据符合研究假设
* 报道内容简短即可（100-200字摘要）

    import { useState } from 'react';
    import { Report } from '../types';
    import DistributionTooltip from './DistributionTooltip';
    
    interface Props {
      report: Report;
    }
    
    export default function FeedItem({ report }: Props) {
      const [showTooltip, setShowTooltip] = useState(false);
    
      return (
        <div 
          className="feed-item"
          onMouseEnter={() => setShowTooltip(true)}
          onMouseLeave={() => setShowTooltip(false)}
        >
          <div className="tags">
            <span className={`tag ${report.source}`}>
              {report.source}·{getStanceLabel(report.stance)}
            </span>
          </div>
    
          <h3>{report.title}</h3>
          <p>{report.summary}</p>
    
          {showTooltip && (
            <DistributionTooltip 
              eventId={report.eventId}
              currentStance={report.stance}
            />
          )}
        </div>
      );
    }

### 6.2 SpectrumBar组件

    interface Props {
      distribution: {
        supportive: number;
        neutral: number;
        opposed: number;
      };
      currentStance: 'supportive' | 'neutral' | 'opposed';
    }
    
    export default function SpectrumBar({ distribution, currentStance }: Props) {
      return (
        <div className="spectrum">
          <div className="labels">
            <span>支持</span>
            <span>中立</span>
            <span>反对</span>
          </div>
    
          <div className="bar">
            <div 
              className="segment support" 
              style={{ width: `${distribution.supportive}%` }}
            >
              {currentStance === 'supportive' && (
                <div className="marker">↑<br/>此报道</div>
              )}
            </div>
    
            <div 
              className="segment neutral" 
              style={{ width: `${distribution.neutral}%` }}
            >
              {currentStance === 'neutral' && (
                <div className="marker">↑<br/>此报道</div>
              )}
            </div>
    
            <div 
              className="segment opposed" 
              style={{ width: `${distribution.opposed}%` }}
            >
              {currentStance === 'opposed' && (
                <div className="marker">↑<br/>此报道</div>
              )}
            </div>
          </div>
    
          <div className="percentages">
            <span>{distribution.supportive}%</span>
            <span>{distribution.neutral}%</span>
            <span>{distribution.opposed}%</span>
          </div>
        </div>
      );
    }

## 7. 设计原则

### 7.1 最小侵入性

**不做**：

* ❌ 弹窗警告"你的看法可能有偏差"
* ❌ 强制阅读分布数据
* ❌ 阻断用户的阅读流程

**而做**：

* ✅ 悬停时轻量提示
* ✅ 用户自主选择是否查看
* ✅ 保持阅读的流畅性

### 7.2 体验式学习

**核心**：让用户自己发现矛盾，而非被动告知

**不加**：

* ❌ "热榜夸大了反对声音"
* ❌ "你的判断可能有误"

**依赖**：

* ✅ 用户的自我意识："咦？怎么只有25%？"
* ✅ 认知失调的自然触发

### 7.3 学术严谨性

**单一变量**：

* 不叠加其他干预（如情绪过滤、去标签）
* 保证测量的是"分布可视化"的效果

**可测量**：

* 记录用户悬停行为（多少用户会查看？）
* 前后测：用户对分布的估计准确度
* A/B对照：有/无分布提示的对比

## 10. 风险与应对

### 风险1：用户不悬停

**表现**：大部分用户滑过Feed，不查看分布**应对**：

* 首次访问引导："试试悬停👆"
* 或：固定在顶部显示一个事件的分布作为示例

### 风险2：分布图太复杂

**表现**：用户看不懂光谱条**应对**：

* 简化设计（已采用最简方案）
* 内测时收集反馈，必要时调整

### 风险3：数据质量

**表现**：AI分类不准确**应对**：

* 人工抽查50篇，校准Prompt
* 必要时手工标注
