# 02-Path-C - 交叉身份可视化

**干预目标**：通过展示评论者的非政治身份标签，让用户自主发现与对立立场者之间的共同身份，软化内外群体边界。

## 📄 核心文档

### [设计规范.md](./设计规范.md)
**必读文档**，包含：
- 核心目标与干预机制（交叉身份 Cross-cutting Identity）
- 触发逻辑（仅依据「对方有无档案」，不做交集检测）
- 四层身份标签体系：
  1. 生活处境（父母、照顾者、退伍军人）
  2. 地域身份（城市、社区）
  3. 职业与教育（职业、校友）
  4. 兴趣与偏好（爱好、文化消费）
- 视觉规范（标签样式、个人叙事展开、标签档案页）

## 🧩 关键设计决策

**为什么不做交集高亮？**
> 若只在有交集时显示标签，「无标签」就成为「无共同点」的负面信号，反而可能强化对立。因此设计为：只要对方有档案就显示，由用户自己发现重合。

**为什么分四层？**
> 按「线下陌生人自然流露程度」排序。第一层（生活处境）的共鸣深度通常高于第四层（兴趣偏好），分层本身即反映系统设计优先级。
