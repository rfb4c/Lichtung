# 01-Path-B - 共识可视化

**干预目标**：通过在评论区前置权威民调数据，打破算法推荐和活跃极端用户制造的「多数错觉」。

## 📄 核心文档

### [设计规范.md](./设计规范.md)
**必读文档**，包含：
- 核心目标与干预机制
- 三层映射架构（报道 → 议题 → 价值观）
- 中层议题库定义（Demo: 枪支管控、堕胎权利、气候政策）
- 可视化组件规范（直方图设计、触发逻辑、交互行为）
- 数据来源要求（Pew Research Center）

### [操作手册.md](./操作手册.md)
**实施指南**，覆盖完整数据管线：
1. 数据源采集（合格数据源的判定标准）
2. 数据清洗与入库（JSON → Supabase）
3. 议题标签系统（报道如何匹配议题）
4. 前端渲染逻辑（何时显示直方图、如何查询数据）

## 🎯 实施检查清单

- [ ] 在 Supabase 创建 `topics` 和 `polling_data` 表
- [ ] 导入 Pew Research 民调数据（3 个议题）
- [ ] 为现有报道打议题标签（手动或通过 News-Agent）
- [ ] 实现前端直方图组件（PollingChart）
- [ ] 实现评论区顶部可视化容器
- [ ] 测试触发逻辑（有议题标签 → 显示；无标签 → 隐藏）

## 📊 数据依赖

| 表名 | 字段示例 | 说明 |
|------|---------|------|
| `topics` | id, name, slug | 议题库（如 `us-gun-control`） |
| `polling_data` | topic_id, labels, values, source | 民调分布数据 |
| `reports` | ..., topic_id | 报道关联议题（FK） |
